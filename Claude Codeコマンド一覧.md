# Claude Code コマンド一覧

作成日: 2026-07-17
このファイルは、Claude Codeで使えるコマンドを後から見返せるようにまとめたリファレンスです。

Claude Codeのコマンドには大きく分けて2種類があります。

- **CLI操作コマンド**: セッション管理・設定など、Claude Code自体を操作するための組み込みコマンド
- **スキル(スラッシュコマンド)**: 特定の作業(コードレビュー、デプロイ確認など)を自動化するために呼び出す機能

---

## 1. CLI操作コマンド

出典: [Claude Code公式ドキュメント (commands.md)](https://code.claude.com/docs/en/commands.md)

### セッション・コンテキスト管理

| コマンド | 説明 |
|---|---|
| `/context` | 現在のコンテキスト使用量を表示する |
| `/compact` | 会話を要約してコンテキストを節約する |
| `/clear` | 新しい会話を開始する(履歴をクリア) |
| `/resume` | 以前の会話を再開する |
| `/branch` | 会話を分岐して作成する |
| `/rewind` | コードと会話を特定のチェックポイントまで巻き戻す |

### 設定・モデル

| コマンド | 説明 |
|---|---|
| `/model` | 使用するAIモデルを切り替える |
| `/effort` | エフォートレベル(low/medium/high等)を設定する |
| `/config` | 設定・テーマ・出力スタイルを調整する |
| `/advisor` | アドバイザーツールの有効/無効を切り替える |

### 開発支援

| コマンド | 説明 |
|---|---|
| `/init` | プロジェクトのCLAUDE.mdを初期化する |
| `/memory` | CLAUDE.md(メモリファイル)を編集する |
| `/diff` | 変更内容をインタラクティブに表示する |
| `/code-review` | 現在の差分をレビューする(正確性・簡素化の観点) |
| `/security-review` | 変更内容のセキュリティレビューを行う |
| `/simplify` | 変更したコードの整理・簡素化レビューを行う |
| `/doctor` | セットアップの診断と修復を行う |
| `/debug` | デバッグログを有効化する |

### 実行・並列処理

| コマンド | 説明 |
|---|---|
| `/batch` | 大規模な変更を並列実行する |
| `/background` | セッションをバックグラウンドで実行する |
| `/fork` | サブエージェントを起動する |
| `/tasks` | バックグラウンドタスクの一覧を表示する |

### その他

| コマンド | 説明 |
|---|---|
| `/plan` | 大規模な変更のためのプランモードに入る |
| `/cd` | 作業ディレクトリを移動する |
| `/btw` | 履歴に記録されない質問をする(補足質問用) |
| `/copy` | 直前のレスポンスをコピーする |
| `/export` | 会話をエクスポートする |
| `/feedback` | バグ報告やフィードバックを送信する |

## 2. スキル(スラッシュコマンド)

このセッションで確認できた利用可能スキル一覧です。プロジェクトや環境によって増減します。

### コードレビュー・品質

| スキル | 説明 |
|---|---|
| `/code-review` | 現在の差分を正確性バグ・簡素化/効率化の観点でレビュー(--comment, --fix オプションあり。ultraで多エージェント・クラウドレビュー) |
| `/simplify` | 変更されたコードを再利用性・簡素化・効率化の観点でレビューし修正まで適用(バグ探しはしない) |
| `/security-review` | 現在のブランチの変更に対するセキュリティレビュー |
| `/review` | GitHubのプルリクエストをレビュー(自分の作業差分には/code-reviewを使用) |
| `/verify` | コード変更が実際に意図通り動くか、実行して確認する |

### 設定・環境

| スキル | 説明 |
|---|---|
| `/update-config` | Claude Codeのハーネス設定(settings.json)を構成。フック・権限・環境変数など |
| `/keybindings-help` | キーボードショートカットのカスタマイズ、キーバインドの変更 |
| `/fewer-permission-prompts` | よく使う読み取り専用コマンドを許可リストに追加し、権限確認を減らす |
| `/init` | プロジェクトを解析してCLAUDE.mdを初期化(コードベースの説明を記載) |

### 可視化・デザイン

| スキル | 説明 |
|---|---|
| `/dataviz` | チャート・グラフ・ダッシュボードなどのデータ可視化を作成する際のデザインガイド |
| `/artifact-design` | Artifacts(公開ページ)作成時のデザイン指針 |
| `/artifact-capabilities` | Artifactsからユーザーのconnector(MCP)を呼び出すランタイム機能の解説 |

### 自動化・スケジューリング

| スキル | 説明 |
|---|---|
| `/loop` | プロンプトやスラッシュコマンドを一定間隔で繰り返し実行する |
| `/schedule` | cronスケジュールで動くクラウドエージェント(ルーティン)を作成・管理する |

### 実行・動作確認

| スキル | 説明 |
|---|---|
| `/run` | プロジェクトのアプリを起動して動作確認・スクリーンショット取得する |

### リファレンス・知識

| スキル | 説明 |
|---|---|
| `/claude-api` | Claude API/Anthropic SDKに関するリファレンス(モデルID、料金、パラメータ、ツール利用など) |

## 補足

- 上記のスキル一覧はセッション・プロジェクト構成に依存するため、環境によって増減する可能性があります。最新の一覧は、Claude Codeとの会話中に表示される利用可能スキル一覧を都度確認してください。
- CLI操作コマンドは公式ドキュメントに基づいていますが、バージョンによって追加・変更される可能性があります。
