# CLAUDE.md

このファイルはClaude Codeがこのプロジェクトで作業する際のガイドです。

## プロジェクト概要

夏祭りで複数の屋台を出すにあたり、数人のメンバーがスマホから売上・帳簿を記録し、
ダッシュボードで状況を確認するためのWebアプリケーション。
一般的な会計ソフトではなく、屋台運営に特化した簡易会計・売上管理ツール。

- 利用対象: 夏祭りで複数屋台を運営する数人(スマホ利用が前提)
- 利用時期: 2026年8月中旬頃の本番(このファイル更新時点で約1ヶ月後)までに実用可能にする
- 現状: コードは未着手。1ヶ月で実用できるMVPを最優先で実装する。

## 主な機能(コア)

- 仕訳入力・帳簿(売上・経費を記録し、帳簿として残す)
- レポート・ダッシュボード(屋台ごと/全体の売上・損益をスマホで可視化)

### 記録する項目

- 売上(商品が売れた都度の入力)
- 経費・仕入れ(食材・備品などの支出)
- 屋台ごとの現金・釣り銭(手元現金の残高管理)
- 在庫・残り個数(商品の残数管理)

## 技術スタック

- バックエンド: Python / Django
- フロントエンド: Djangoテンプレート + スマホ最適化のレスポンシブUI(短期開発のため別フレームワークは使わない方針)
- DB: 本番はPostgreSQL / 開発はSQLite(`DATABASE_URL` 環境変数の有無で自動切替)
- デプロイ先: Render(`render.yaml` Blueprint。web=gunicorn、DB=PostgreSQL)
- 静的配信: WhiteNoise(collectstatic + 圧縮マニフェスト方式)
- 本番設定は環境変数で制御(手順は [docs/deploy-render.md](docs/deploy-render.md) を参照)

## 開発方針

- Djangoの標準構成(プロジェクト直下に `manage.py`、アプリ単位でディレクトリ分割)に従う
- 金額計算には浮動小数点数(float)を使わず `Decimal` を使用する
- スマホでの入力しやすさ・数字の視認性・誤操作防止を最優先する
- 1ヶ月で本番運用するため、機能を絞ったMVPから作り、動くものを早く用意する

## 認証・アクセス方針

- ログイン機能は設けず、「URLを知っていれば使える」方式とする(数人での手軽な共有を優先)
- ただしURL漏洩=誰でも売上を書き換えられるリスクがあるため、推測されにくいトークン付きURL
  (例: `/s/<ランダムトークン>/`)でアクセスを保護する
- 屋台ごとにURL/トークンを分けることを検討する
